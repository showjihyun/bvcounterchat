# 메트릭 베이스라인

목적: "하네스가 먹혔는가"를 감이 아니라 숫자로 회고하기.
수집은 전부 기존 산출물(git·CI·로그)에서 — 별도 인프라 없음.

> 이 프로젝트는 아직 코드가 없다. M1~M6·M8~M9의 베이스라인은 "미측정"이
> 정직한 값이다 — 측정할 PR·커밋·틱·프레임 자체가 존재하지 않는다.
> M7(골든 케이스 수)만 지금 이 문서와 함께 실제 값을 갖는다.

## 기록 규칙

- 주 1회(회고 때) "현재" 열만 갱신한다. 5분 이상 걸리면 지표를 줄인다.
- M1~M3이 드리프트 직접 지표. M4~M6은 부작용 감시(하네스가 속도를
  죽이고 있지 않은지). M7은 학습 루프 작동 여부.
- M8~M9는 게임 특화 성능 지표 — 계측 인프라(틱 계측, 프레임 타이밍 수집)가
  구축되기 전까지 수집 불가하다. 서버 틱 루프(RQ-60)·클라이언트 렌더 루프의
  최초 수직 슬라이스 구현과 함께 계측 코드를 반드시 같이 넣는다(사후 추가는
  누락되기 쉽다).
- 회고 때: 베이스라인 대비 변화 + "지운 지표와 이유"까지가 L&L 산출물.
